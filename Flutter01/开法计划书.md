# 1. 项目概述与目标

**目标**：实现一个可在 Android / iOS / Web（可选）上运行的 Flutter 可复用组件 `VoiceWaveWidget`，接收录音/音量数据并以垂直圆角矩形条（bar）实时显示音量动画。动画参考 iPhone 语音备忘录，新的数据从右侧进入，整体从右向左移动，条高在 **12dp — 58dp** 之间，条间距 **6dp**。

**主要特性**：

- 实时绘制音量条（平滑动画、峰值回退、抖动抑制）
- 可配置样式（颜色、最大/最小高度、条宽、间距、圆角）
- 支持两种输入模式：实时流（每帧或每 N ms 的音量值）和批量样本（历史数据）
- 支持暂停/回放模式（可用于录音回放时渲染）
- 低 CPU & GPU 占用、保持高帧率

------

# 2. 需求规格（功能性 & 非功能性）

## 功能需求

1. **绘制规则**
   - 每条为垂直圆角矩形（可左右对称或单侧），条高在 12dp — 58dp。
   - 条间距为 6dp（固定或可配置）。
   - 新数据从右侧插入，所有条向左移动（连续滚动效果）。
2. **动画**
   - 插入时有平滑增长动画（ease-out），衰减时有轻微延迟（peak hold），整体平滑过渡。
   - 支持帧率控制（目标 60fps，允许 120fps 如果设备支持）。
3. **输入**
   - 支持传入的音量值为 0.0 — 1.0（或 dB），组件内部映射到高度区间。
   - 提供 API `addSample(double amplitude)` / `addSamples(List<double> amplitudes)`。
4. **样式配置**
   - 最小/最大高度、条宽、间距、颜色、圆角半径、背景、是否镜像（两侧对称）等可配置。
5. **生命周期**
   - 支持 start/stop/clear/pause/resume。
6. **平台权限与录音**
   - 仅提供组件，实际录音由上层负责（或提供示例接入方案）。示例使用 `flutter_sound` 或 `mic_stream`（可替换）。

## 非功能需求

- **性能**：尽量使用 `CustomPainter` + `RepaintBoundary`，避免大量 widget 重建；单次 repaint 复杂度 O(Nbars)；内存占用低。
- **可测试性**：提供单元测试（绘制逻辑映射）、集成测试（与模拟音量流）。
- **可配置/可复用**：支持在不同大小容器内自适配（height 限制 12—58dp 由调用方约束或组件内 clamp）。
- **无侵入性**：不会自行请求权限（录音权限由应用层处理）；但在示例里展示如何申请权限。

# 4. 架构设计（组件内部结构）

```
VoiceWaveWidget (StatefulWidget)
├─ VoiceWaveController (exposes addSample/clear/pause/resume)
├─ _VoiceWaveState
│  ├─ List<double> samples  // 环形缓冲或队列，长度 = visibleBars + buffer
│  ├─ AnimationController / Ticker  // 驱动“从右到左”平移
│  └─ CustomPaint(painter: VoiceWavePainter(samples, styleConfig, animationValue))
```

## 数据流

- 上层录音模块 -> 归一化音量值（0.0-1.0） -> `VoiceWaveController.addSample(value)` -> 将样本 push 到 samples 队列 -> 触发 repaint（或在动画 tick 中平滑插值）

## 绘制策略

- 组件维护一个固定数量的“格/条”（`barCount`），每个格对应时间窗口（例如每 40ms 一个格，实际时间由上层控制）。
- 使用 `CustomPainter` 在 `paint()` 中按 samples 绘制圆角矩形。
- 平移动画：通过 `animationValue`（0..1）控制所有条向左偏移 `barWidth + spacing` 的插值。
- 平滑：在插入 sample 时进行线性/贝塞尔插值或使用低通滤波器（alpha 平滑）以避免抖动。

# 6. 关键实现要点（细节说明）

1. **高度映射**
   - `height = minBarHeight + (maxBarHeight - minBarHeight) * clamp(amplitude, 0, 1)`
   - 对于 dB 输入，可先转换：`amplitude = (db - minDb) / (maxDb - minDb)` 并 clamp
2. **数量与间隔**
   - `barCount` = floor((widgetWidth + spacing) / (barWidth + spacing))，在 build 时计算动态适配
3. **动画**
   - 使用 `Ticker` 每帧推进 `animationValue`，当有新样本入列时，触发一次 `animationController.forward(from:0)`，并在 tick 中 `setState()` 触发 `CustomPainter.paint`。
   - 为避免频繁 `setState`，只在 `animation` tick 触发 `repaint`：通过 `CustomPainter` 的 `repaint` 参数传入 `Listenable`（AnimationController）
4. **性能优化**
   - 将绘制集中在 `CustomPainter`，避免每个 bar 使用一个 widget。
   - 使用 `RepaintBoundary` 包裹组件，避免上层重绘传导。
   - 限制 `barCount`（例如 40~120，取决设备），避免太大导致每帧绘制过多。
   - 使用 `Canvas.drawRRect` 批量绘制，避免额外分配。
5. **平滑 & 减抖**
   - 使用一阶低通滤波 `y[n] = alpha * x[n] + (1-alpha) * y[n-1]`（alpha 控制平滑）。
   - 对极端 spike（突发噪音），增加峰值保持与快速衰减（peak hold + decay）逻辑。
6. **回放模式**
   - 对回放，可一次性加载历史 amplitudes，并以固定速度播放（控制插入速率）。

# 7. 示例代码片段（Widget + Painter 核心逻辑骨架）

> 下面是简化版示例（供开发快速启动），实际工程中要补充边界检查、资源释放与配置项。

```
class VoiceWaveWidget extends StatefulWidget {
  final VoiceWaveController controller;
  // ... config fields ...
  const VoiceWaveWidget({required this.controller, ...});
  @override
  _VoiceWaveState createState() => _VoiceWaveState();
}

class _VoiceWaveState extends State<VoiceWaveWidget> with SingleTickerProviderStateMixin {
  late AnimationController _anim;
  List<double> samples = []; // newest at end
  @override
  void initState() {
    super.initState();
    _anim = AnimationController(vsync: this, duration: Duration(milliseconds: 300))
      ..addListener(() => setState(() {}));
    widget.controller._bind(this);
  }
  void _pushSample(double amp) {
    samples.add(amp);
    if (samples.length > widget.barCount) samples.removeAt(0);
    _anim.forward(from: 0.0);
  }
  @override
  Widget build(BuildContext context) {
    return RepaintBoundary(
      child: CustomPaint(
        size: Size.infinite,
        painter: VoiceWavePainter(
          samples: samples,
          animationValue: _anim.value,
          // ...config
        ),
      ),
    );
  }
  @override
  void dispose() {
    _anim.dispose();
    super.dispose();
  }
}

class VoiceWavePainter extends CustomPainter {
  final List<double> samples;
  final double animationValue;
  // ...config...
  VoiceWavePainter({required this.samples, required this.animationValue, ...}): super();
  @override
  void paint(Canvas canvas, Size size) {
    final paint = Paint()..color = barColor;
    final totalWidth = size.width;
    final barSlot = barWidth + spacing;
    final baseRight = totalWidth; // bars anchored from right
    for (int i = 0; i < samples.length; i++) {
      final indexFromRight = samples.length - 1 - i;
      final x = baseRight - (indexFromRight + animationValue) * barSlot - barWidth;
      final h = minBarHeight + (maxBarHeight - minBarHeight) * samples[i];
      final y = (size.height - h) / 2; // vertical center
      final rect = RRect.fromRectAndRadius(Rect.fromLTWH(x, y, barWidth, h), Radius.circular(barRadius));
      canvas.drawRRect(rect, paint);
    }
  }
  @override
  bool shouldRepaint(covariant VoiceWavePainter old) => true;
}
```

# 10. 风险点与缓解方案

- **风险**：频繁 repaint 导致性能问题
   **缓解**：使用 `CustomPainter` + `AnimationController` 驱动，限流样本速率，使用 `RepaintBoundary`。
- **风险**：录音采样与 UI 不同步（不同时间基准）
   **缓解**：约定样本时间窗口（例如每 40ms 为一格），若采样率不稳定则在 controller 内做聚合/重采样。
- **风险**：低端设备上性能不足
   **缓解**：提供 `lowPerformanceMode`（降低 barCount、降低动画帧率、简化绘制）。
- **风险**：包依赖或平台权限变动
   **缓解**：将录音模块作为示例，组件本身只依赖 Flutter 核心，避免强依赖第三方包。
